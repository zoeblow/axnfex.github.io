UC浏览器返回上一页快照BUG

发表于 2016-12-10 by 水三千   |   分类于 踩坑记录   |  

嗯，那是一个阳光明媚的午后，我正愉悦的撸着代码，突然测试提了个BUG过来。

“UC上跳转到任何页面时，立刻浏览器自带的返回，再点前进，页面渲染出错，Ajax请求出错。”

当时我的内心是崩溃的，又是UC这个**（手动和谐），没办法，改呗。
先复现一波:

当时我的第一反应就是会不会是后台请求数据时间较长，Ajax请求异常，导致页面渲染出错，于是调试了一番发现，在请求后台数据的时候，确实请求时间过长。

与后台沟通了下，修改了请求数据库的方式，减少了请求的时间，优化了渲染DOM的操作。再测试，然并卵….

后来仔细想了下，这个BUG会不会是异步请求导致的？由于请求没有结束就跳转了页面，再返回的时候页面的Ajax请求出错？？ 由于BUG只能在手机上复现，chrome浏览器上的模拟器复现不了，在手机上调试一直是一个大问题，但是！！！我们有老司机啊，老司机带你超神带你飞，eruda.js，让你可以在手机上调试你的代码，起飞。调试后发现，果然

发现返回页面后Ajax请求状态码直接为0，http协议里可是没这个状态码的，后来找资料，百度，google，找到了一个这个XMLHttpRequest的说明：http://www.w3.org/TR/XMLHttpRequest/ 

The status attribute must return the result of running these steps:
status的值一定会返回运行这些步骤的结果。
  • If the state is UNSENT or OPENED, return 0.（如果状态是UNSENT或者OPENED，返回0）
  • If the error flag is set, return 0.（如果错误标签被设置，返回0）
  • Return the HTTP status code.（返回HTTP状态码）

既然找到原因了，解决也就好解决了，判断状态码为0的时候刷新一下页面就好了，这么简单搞了一个下午，UC浏览器果然是个坑，巨坑，嚎哭深渊。
