创建团队博文精简教程

发表于 2016-12-28 by 阿牛   |   分类于 技术分享   |  

很多团队都有自己的博客，里面记录了每个创作人的所闻所想，可以是技术方面的，也可以是技术之外的，只要是你想到的并且想记录下来的，就可以为此写一篇博文。写作不仅可以给自己做笔记，提升自己的表达以及总结能力，也可以分享给团队成员，也许别人遇到了和你同样的问题，看了你分享的文章就解决了，所以鼓励大伙平时多写写博文。
我们爱信诺前端团队的博客是利用hexo+github搭建的，其中hexo是在本地生成静态文件然后发布到github上，在完成这些操作之前需要完成以下几件事：

配置环境

1.安装nodejs
建议安装稳定版本，安装过程无需特殊配置，一路下一步即可。
默认nodejs是用npm安装第三方包的，由于服务器是在国外所以下载的非常慢，因此需要将服务器切换到国内，这里用到了淘宝NPM镜像。桌面右键点击Git Bash Here，输入下面代码，回车等它安装完成即可，这样以后导第三方包用cnpm代替npm。
npm install -g cnpm --registry=https://registry.npm.taobao.org
2.安装hexo
命令行输入下面代码，回车安装。
cnpm install hexo-cli -g
3.安装svn
这个软件简称小乌龟，下载和提交hexo配置文件用的。
4.安装git
发布博文必须用到git，下载最新版一路下一步即可。
安装步骤中Git Bash Here建议勾选，它会在windows右键菜单上增加快捷按钮，用于命令行目录定位，不需要再用cmd敲一大堆代码定位了。

5.创建公私钥
咱们博文提交到github不是通过https的，而是ssh，因此需要本地创建公私钥后将公钥提供给博主或者老司机，待配置后才有权限提交，如何创建的请看ssh-key配置多个远程仓库。

部署发布

1.下载hexo配置文件
svn地址在前端群公告里，大家自行下载，或者点击查看，将svn检出到任意目录。
配置文件是托管到码云上的，没有的需要注册，svn帐号密码就是码云账号。
这里和网上的hexo搭建博客教程是有区别的，他们的是在博客目录下先敲命令hexo init下载配置文件，咱们是用svn下载配置文件，这样整个团队的配置文件都统一。如果按网上的操作，你发布到github的文章只有你自己的，别人的文章就会被清空。所以大伙在发布文章之前一定要先更新，而自己的文章也别忘记提交了，其中打箭头的不要提交了！

2.初始化
打开配置文件检出目录，右键Git Bash Here，输入命令安装包依赖：
cnpm install
此操作会在目录下生成node_modules文件夹
3.熟悉命令
一切命令操作都是在博客根目录（检出目录）完成的
hexo clean
清理缓存文件
hexo generate / hexo g
生成静态文件，此操作在目录下会生成public文件夹和db.json
hexo server / hexo s
启动服务，访问地址http://localhost:4000
hexo deploy / hexo d
发布到github，此操作在在目录下会生成.deploy_git文件夹
hexo也提供了其它命令，例如新建文章，大家自己去官网查看api。
其实发布文章很简单，写好文章后只需要敲2个命令就可以了：
hexo g > hexo d
4.写文章
文章是通过markdown编写的，语法请自行去熟悉，网上也有很多软件，大家根据个人喜好使用。博主使用的是Atom，不过需要安装一些插件配合才爽，自己也遇到了一些坑希望能帮助大家：Atom编辑器使用markdown走过的坑。
所有文章都是放在博客根目录/source/_posts/这个下面，如果博文中需要使用图片的话，需要在文章目录下新建一个文件夹必须和文章的文件名相同。

总结

写文章不是必须的，没人强迫大家写，只是建议大伙多做分享，自己学到的知识也可以分享给同事，让团队共同成长！
最后再强调一遍：发布文章之前一定要先更新，而自己的文章也别忘记提交了，结构图中打箭头的不要提交了！
